## Summary

Comprehensive browser automation testing of ALL user interactions in the AutoVoice web interface via VNC viewport, using keyboard-based navigation (Tab + Enter) since mouse clicks don't work reliably on the headless X server.

## Context

AutoVoice is a GPU-accelerated singing voice conversion system with a React frontend. The user wants to WATCH tests execute in real-time via VNC at http://192.168.1.64:16080/vnc.html. The tests must be visible to the user.

## Environment

- **Backend:** Port 10600 (systemd service `autovoice.service`)
- **Frontend:** https://autovoice.giggadev.com (Apache reverse proxy)
- **VNC:** http://192.168.1.64:16080/vnc.html
- **X Display:** :99 (Xvfb 1920x1080x24)
- **Browser:** Chromium (launched directly, MCP playwright tools don't work on ARM64)
- **Automation:** xdotool with --window flag for keyboard input

## Technical Constraints

1. **ARM64 Platform:** Jetson Thor - Chromium-based MCP tools (playwright-headed, puppeteer) don't work
2. **No Window Manager:** Xvfb runs without window manager, mouse clicks unreliable
3. **Solution:** Use keyboard navigation (Tab to navigate, Enter to activate, Ctrl+L for URL bar)

## Technical Notes

### Keyboard Navigation Pattern
```bash
# Get window ID
WID=$(DISPLAY=:99 xdotool search --name "chromium" | head -1)

# Navigate to URL
DISPLAY=:99 xdotool key --window $WID ctrl+l
DISPLAY=:99 xdotool type --window $WID "https://autovoice.giggadev.com/profiles"
DISPLAY=:99 xdotool key --window $WID Return

# Tab to element and activate
DISPLAY=:99 xdotool key --window $WID Tab Tab Tab  # Navigate to button
DISPLAY=:99 xdotool key --window $WID Return        # Click/activate
```
