## Summary

Complete frontend integration to expose all new backend capabilities to users via the web interface. This track coordinates with three parallel tracks to ensure the UI reflects all new features as they're implemented.

## Context

Three parallel tracks are adding backend capabilities that need frontend exposure:

1. **training-inference-integration**: API endpoints for trained model selection
   - GET /api/v1/profiles/{id}/model - Check if profile has trained model
   - POST /api/v1/convert/song with profile_id - Convert with specific profile
   - Profile responses include `has_trained_model` flag

2. **sota-dual-pipeline**: Pipeline selection and benchmarks
   - Realtime pipeline: RTF 0.475, for karaoke
   - Quality pipeline: RTF 1.981, for studio
   - Combined pipeline: Seed-VC + HQ-SVC super-resolution
   - Benchmark results available for display

3. **sota-innovations**: New pipeline types
   - Shortcut flow matching: 2-step inference (2.83x speedup)
   - MeanVC streaming: Single-step realtime
   - New pipeline types to add to selector

## User Stories

1. As a user, I want to see which voice profiles have trained models so I can use them for conversion.
2. As a user, I want to select different pipeline types (realtime/quality/shortcut) based on my needs.
3. As a user, I want to see conversion progress and quality metrics after conversion.
4. As a user, I want the karaoke page to use my selected voice profile and pipeline in real-time.
5. As a user, I want to see benchmark comparisons to understand quality/speed tradeoffs.
