# Specification: Speaker Diarization & Auto-Profile Creation

**Track ID:** speaker-diarization_20260130
**Type:** Feature
**Created:** 2026-01-30
**Status:** Draft

## Summary

Implement automatic speaker detection to identify different singers in audio files. Parse YouTube metadata to identify featured artists, use SOTA speaker diarization (pyannote.audio/ECAPA-TDNN) to segment audio by singer, filter target artist's training data to only include their vocals, and automatically create profiles for detected collaborating artists.

## Context

AutoVoice needs high-quality training data for voice models. When downloading songs from YouTube, many tracks include featured artists, harmonies, or duets. Currently, all vocals get mixed into the target artist's training data, degrading model quality. This feature ensures training data purity by detecting and separating different singers.

## User Story

As a music producer using AutoVoice, I want the system to automatically detect and separate different singers in my training audio so that each voice profile only contains that artist's vocals, improving model quality and automatically building a library of artist profiles.

## Acceptance Criteria

- [ ] YouTube downloads detect featured artists from video title/description (ft., vs., with, &)
- [ ] Speaker diarization segments audio by singer using SOTA embeddings
- [ ] Segments are matched to existing profiles by speaker similarity or new profiles created
- [ ] Target artist's training data contains only their vocals (other singers filtered out)
- [ ] Web UI shows detected artists and allows review/correction of segment assignments
- [ ] Supports rap/speech detection in addition to singing
- [ ] Handles multiple singers (no arbitrary limit on distinct singers)
- [ ] Frontend updated with diarization results display and segment assignment UI
- [ ] Tests cover diarization pipeline, profile matching, and UI components

None - all use cases including rap, speech, real-time, and unlimited singers are in scope.
