## Summary

Comprehensive LoRA lifecycle management including automatic training, retraining, voice identification, profile matching, and quality monitoring.

## Core Capabilities

### 1. LoRA Audit on Project Completion
- Examine all existing LoRAs in voice_profiles/*/adapters/
- Check training freshness (flag if >30 days old)
- Validate quality metrics (speaker_similarity, MCD)
- Identify profiles needing training/retraining

### 2. Voice Identification Pipeline
- Listen to audio and identify vocalist
- Match against existing profile embeddings (WavLM)
- Threshold-based matching (default 0.85 cosine similarity)
- Auto-route separated vocals to correct profiles

### 3. Automatic Profile Creation
- Create new profile for unknown speakers
- Generate speaker embedding from first sample
- Initialize metadata with creation timestamp
- Queue for LoRA training when samples >= 5

### 4. Auto-Training Logic
**Train New LoRA When:**
- Profile has >= 5 samples
- No existing LoRA adapter
- Profile flagged "needs_training"

**Retrain Existing LoRA When:**
- >3 new samples added since last training
- Quality degradation (speaker_sim drops below threshold)
- LoRA >30 days old with new samples available
- User explicitly requests retrain

### 5. Vocalist Separation Strategy
For multi-artist tracks:
1. Demucs vocal separation
2. Pyannote diarization on separated vocals
3. WavLM embedding extraction per segment
4. Cluster by speaker similarity
5. Match clusters to known profiles
6. Create profiles for new artists
7. Auto-assign segments to profiles

### 6. Quality Monitoring
- Track speaker similarity over time
- Monitor MCD for each profile's conversions
- Alert on quality degradation
- Auto-trigger retrain for declining quality

## Ultimate Goal: Song Conversion

The LoRA lifecycle management enables the ultimate project goal:

### Song Conversion Mode
Convert one artist's voice (e.g., William) to sing another artist's song (e.g., Connor) **exactly the way the original artist sang it**:
- **Pitch Correct**: Match the original pitch contour exactly
- **Singing Abilities Matched**: Transfer vibrato, dynamics, articulation
- **Synced to Instrumental**: Perfect alignment with backing track
- **Voice Identity**: Target artist's voice timbre/identity preserved

### Live Karaoke Mode
Auto-morph user's live singing to match original artist's performance:
- **Real-time Conversion**: Low-latency voice morphing (<100ms)
- **Multiple Audio Outputs**:
  - **Audience Output**: Converted vocals + instrumental for speakers
  - **Headphone Output**: Original song for user to sing along with
- **Practice Mode**: User hears original to match pitch/timing while audience hears converted voice

### Quality Requirements
- Speaker similarity >= 0.85 for trained LoRAs
- MCD <= 4.5 for high-quality conversion
- RTF < 0.3 for real-time processing
